1. Executive Summary  
VERLOX Staffing is a registered UK recruitment business with an established client base, operational infrastructure, and compliance framework. This business plan outlines the strategic launch and operational model for VERLOX Staffing's Virtual Assistant (VA) Division — a managed, recurring-revenue service line that sits alongside the existing placement recruitment business.  
The VA Division will source, vet, and manage virtual assistants — primarily from the Philippines or African and South Africa — and provide them to UK businesses on a flexible retainer or hourly basis. VERLOX Staffing acts as the managed service provider: the single point of accountability for quality, performance, compliance, and client satisfaction.  
The division leverages VERLOX Staffing's existing legal entity, insurance, client relationships, and operational knowledge — dramatically reducing startup risk and cost compared to launching a standalone agency.

2.1 Company Overview  
VERLOX Staffing Ltd is a UK-registered recruitment business. The VA Division is a new service line operating under the same legal entity, trading as VERLOX Staffing — VA Division (or VERLOX VA for marketing purposes).

2.3 Value Proposition  
For UK clients, VERLOX VA eliminates the burden of international hiring: no sourcing, no vetting, no payroll complexity, no GDPR risk management, no performance management. They simply receive a skilled, managed assistant for a predictable monthly fee.  
For VAs, VERLOX provides consistent, well-paying UK work with reliable payment in USD/GBP via Wise, structured task management, and professional development.

The global VA market was valued at approximately $4.12 billion in 2023 and is projected to grow at a CAGR of over 20% through 2030. The UK represents one of the strongest adoption markets, particularly among SMEs and professional services firms.

MIS TECHNICAL SPECIFICATION  
VERLOX Staffing — VA Division Management Information System  
Full Stack Developer Specification | Ready to Build  
1. Project Overview  
This document is the complete technical specification for the VERLOX VA Division MIS — a purpose-built management dashboard that enables VERLOX to manage clients, virtual assistants, contracts, billing, time tracking, and profitability from a single platform.  
The system has three user roles: Admin (VERLOX director/ops), Client (read-only portal), and VA (time logging and invoice submission). The primary interface is a web dashboard.  
1. Technology Stack  
2. System Modules & Feature Specification  
MODULE 1: Authentication & Roles  
Three roles: ADMIN (full access), CLIENT (read-only portal — view own invoices, time logs, NPS), VA (submit time logs, upload invoices, view own payment history).  
Implementation: Auth0 or Supabase Auth with JWT tokens. Role stored in DB. Protected API routes with role middleware.  
Features: Email/password login, password reset, session management (Redis), 2FA optional for Admin.  
MODULE 2: Client Management (CRM)  
Core module. Lists all clients with status (Active, Onboarding, Paused, Churned). Each client record shows: company name, contacts, tier, contract dates, assigned VA, total revenue to date, NPS score, outstanding invoices.  
Actions: Add new client, edit details, change tier, change assigned VA, view full invoice history, log communication notes, attach documents, view time usage this month.  
Key views: Client list (sortable/filterable), client detail page, contract overview panel.  
MODULE 3: VA Management  
Manages the VA contractor roster. Each VA profile: name, country, timezone, specialism tags, hourly rate, availability status (Available, Assigned, Unavailable), performance score (1–5 stars, updated monthly), current clients, total earnings paid to date.  
Actions: Add VA, edit profile, change availability, view time logs, process payment, deactivate.  
Matching view: Filter VAs by specialism, timezone, and availability to find best match for incoming client.  
MODULE 4: Contracts  
Links client to VA with commercial terms. Each contract: client_id, va_id, tier, hours per month, client rate (£/month), VA rate (£/hour), start date, end date (or rolling), status (active/ended/paused).  
Auto-calculates: client monthly invoice amount, VA monthly payment, gross profit, margin %.  
Actions: Create contract, view/edit, terminate, renew. Auto-alerts 30 days before contract end.  
MODULE 5: Time Tracking  
VA-facing interface for logging hours. Each log entry: VA, client, date, hours, task description. Submitted at end of each working day or week.  
Admin view: Approve or flag time logs. Approved logs feed into invoice and payment calculations.  
Client view (portal): Can see time logs against their account — hours used vs contracted.  
Auto-alert to admin when client hits 80% of monthly hours.  
MODULE 6: Billing & Invoicing  
Monthly auto-generation of client invoices based on: contracted retainer + any approved overage hours x overage rate. Invoice includes breakdown of hours and tasks.  
Stripe integration: Send invoice via Stripe, accept card/bank payment, auto-record payment date.  
Invoice statuses: Draft, Sent, Paid, Overdue. Overdue invoices trigger auto-chase email at 7 days and alert at 14 days.  
Admin can create manual invoices and credit notes.  
MODULE 7: VA Payments  
Monthly payment batch: System calculates each VA's payment based on approved time logs x VA rate. Shows breakdown per VA: hours, rate, total due, currency.  
Wise API integration: Admin reviews batch, approves, and initiates bulk payment via Wise. System records transfer ID and confirms payment.  
VA can see their payment history in their portal.  
MODULE 8: Margin & Profitability Dashboard  
The core financial intelligence module. For each contract/client: shows revenue, VA cost, gross profit, margin %. For the business overall: blended margin, monthly P&L, revenue by tier, revenue trend chart.  
Key charts: Monthly revenue bar chart, margin trend line, revenue by client pie chart, top/bottom clients by margin.  
Export: Monthly P&L as CSV or PDF for accounting.  
MODULE 9: Main Dashboard (Home)  
Executive summary view. Panels:  
Row 1 (KPI cards): Revenue MTD | Active Contracts | Gross Margin % | Outstanding Invoices  
Row 2 (KPI cards): VA Utilisation % | Avg NPS Score | New Clients MTD | Pending VA Payments  
Row 3: Revenue chart (12 months bar) | Margin trend (line chart)  
Row 4: Client health table (client name, NPS, hours used %, invoice status) | Upcoming tasks (invoice due, contract renewals, reviews)  
MODULE 10: Document Vault  
S3-backed document storage. Each client and VA has a document section. File types: PDF, DOCX, images. Document types: Contract, NDA, DPA, Invoice, Onboarding form, Other.  
Upload, view, download, delete. All uploads tagged with date and uploader. Signed contracts auto-linked from HelloSign webhook.  
MODULE 11: NPS & Client Feedback  
Monthly automated email to active clients (via SendGrid) with a simple 1–10 NPS question. Responses captured via API and stored. Dashboard shows current avg NPS, trend, and flags clients scoring below 7.  
MODULE 12: Communication Log  
Simple CRM log. Per client or VA: log a note, email, or call. Entries show: type, summary, date, logged by. Helps the whole team see client history at a glance.  
1. Database Schema  
PostgreSQL. The following Prisma schema defines all core tables:  
-- CLIENTS  
model Client {  
id             String    @id @default(uuid())  
companyName    String  
contactName    String  
email          String    @unique  
phone          String?  
tier           Tier      // STARTER | GROWTH | PROFESSIONAL | ENTERPRISE | ADHOC  
status         ClientStatus // ONBOARDING | ACTIVE | PAUSED | CHURNED  
monthlyRate    Decimal   // GBP — client billing rate  
hoursIncluded  Int       // contracted hours per month  
overageRate    Decimal   // GBP per hour over contracted amount  
contractStart  DateTime  
contractEnd    DateTime?  
stripeId       String?   // Stripe customer ID  
createdAt      DateTime  @default(now())  
contracts      Contract[]  
invoices       Invoice[]  
npsResponses   NpsResponse[]  
}  
-- VIRTUAL ASSISTANTS  
model VirtualAssistant {  
id             String    @id @default(uuid())  
fullName       String  
email          String    @unique  
country        String  
timezone       String  
hourlyRateGBP  Decimal  
skills         String[]  // array of skill tags  
availability   VAStatus  // AVAILABLE | ASSIGNED | UNAVAILABLE  
performScore   Float?    // 1.0 to 5.0  
wiseEmail      String?   // payment destination  
currency       String    @default("GBP")  
contracts      Contract[]  
timeLogs       TimeLog[]  
}  
Additional models: Contract, TimeLog, Invoice, VAPayment, MarginLedger, Document, NpsResponse, CommunicationLog. Full Prisma schema available as a separate file (schema.prisma).
